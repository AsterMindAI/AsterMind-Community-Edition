Why TypeScript instead of plain JS?

One reason, then the others: errors at edit time, not at runtime.

JavaScript is happy to run this:

const labels = ['cat', 'dog'];
labels.push(42);          // runs fine
console.log(labels[2].toUpperCase());  // 💥 crashes here

TypeScript catches the bad push on the line where it happens, before you ever run the code. That moment-of-correction is what makes a beginner faster.

Other reasons (lower priority for today): autocomplete that actually knows what's available; safe refactors across files; documentation that lives in the type signatures.

What npm actually does

Three things, in this order:

  1. Reads package.json to learn which third-party packages this project depends on.
  2. Downloads them into node_modules/ (a directory you never edit and never commit).
  3. Lets you run scripts defined in package.json with npm run <name>.

We have 0 runtime dependencies in this repo (look at the "dependencies" field — it's empty). The only packages we install are dev tools: rollup, typescript, vitest, vite.

That's deliberate. Less to break, less attack surface, no surprise updates.

From src/ to dist/

The library ships as JavaScript, not TypeScript. There's a build step in between.

src/        →   rollup -c                →   dist/astermind.umd.js
                rollup -c                    dist/astermind.esm.js
            →   tsc -p tsconfig.types.json   dist/index.d.ts

Two outputs from rollup (UMD + ESM bundles), one output from tsc (the type declarations .d.ts). The lesson decks load dist/astermind.umd.js via a <script> tag — that's why the next two lessons can write window.astermind.IntentClassifier without an import.

You don't have to run any of this manually. npm run build does it all.

How npm test works

vitest finds every *.test.ts file under tests/ and runs it. A test file looks like: 

import { describe, it, expect } from 'vitest';
import { IntentClassifier } from '../src/index';

describe('IntentClassifier', () => {
  it('predicts greetings', () => {
    const clf = new IntentClassifier({ /* ... */ });
    clf.train([{ text: 'hello', label: 'greeting' }]);
    const result = clf.predict('hi', 1);
    expect(result[0].label).toBe('greeting');
  });
});

describe groups, it declares one test, expect asserts. If expect sees something it didn't expect, the test fails and the whole run goes red.

You-Try 1 — clone, install, test

Open a terminal. Run these four commands:

git clone git@github.com:AsterMindAI/AsterMind-Community-Edition.git astermind-l01
cd astermind-l01
npm install
npm test

Pass condition: vitest prints ≥106 passing, 0 failing. Time yourself — should be under 5 minutes.

If git clone fails because of SSH, swap to the HTTPS URL: https://github.com/AsterMindAI/AsterMind-Community-Edition.git. If npm install fails, check node --version is 18+.
