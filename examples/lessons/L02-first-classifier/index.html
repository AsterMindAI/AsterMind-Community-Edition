Classification, plain

You give the model labelled examples:

"hello"      → english
"bonjour"    → french
"good morning" → english
"salut"      → french
...

It learns. Then you give it a new string it has never seen, and it tells you which label probably fits — with a confidence number.

predict("hi friend") → [{ label: "english", prob: 0.91 }, ...]

That's it. That's classification. Everything else this lesson covers is the wiring around that core idea.

Meet IntentClassifier

It's the simplest text classifier in the library. Three lines to build one:

const { IntentClassifier } = window.astermind;

const clf = new IntentClassifier({
  categories: ['english', 'french'],
  hiddenUnits: 32,
  useTokenizer: true,
  activation: 'relu',
});

Underneath it's an Extreme Learning Machine — what L00 covered. You don't need to remember how it works to use it. You'll be able to swap models in later lessons; the API is the same.

Train + predict in code

// Build it (already shown).
const clf = new IntentClassifier({ /* ... */ });

// Train it.
clf.train([
  { text: 'hello',     label: 'english' },
  { text: 'bonjour',   label: 'french'  },
  // ... 18 more
]);

// Predict.
const result = clf.predict('hi friend', /* topK = */ 2);
// result === [
//   { label: 'english', prob: 0.91 },
//   { label: 'french',  prob: 0.09 },
// ]

That's the whole API for today. predict returns a sorted list (most-likely first); topK controls how many you get back.

Two accuracies, only one matters

  • Training accuracy — how the model does on examples it has seen. Should be high. If it isn't, something's broken.
  • Held-out accuracy — how the model does on examples it has never seen during training. This is the number that matters.

If training accuracy is 100% but held-out accuracy is 50%, the model has memorized the training set and learned nothing transferable. That's called overfitting, and you'll see lots of it as you experiment.

For our 20-example greeting dataset, ≥80% held-out accuracy is the bar.

Things to notice

  • How long did training actually take? (It's milliseconds. ELMs solve in closed form — no gradient descent.)
  • Re-click Train without changing anything. Does the held-out accuracy match the previous run? Why might it not?
  • After adding Spanish, is held-out accuracy higher, lower, or the same? Was that what you expected?
  • What happens when you type a word that's in none of the languages the model knows? (Try "おはよう".)
